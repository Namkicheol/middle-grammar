# 워크시트 신규 제작 워크플로우

## 저작권: 문장 변형 규칙 (필수 · 강제)

교과서·교재 예문 참고 시 **이름·대명사만 교체는 불충분**. 주어 + 목적어/부사구 포함 **최소 2개 이상** 바꿔야 한다.

- **인명 교체**: Jenny → Mia, Kevin → Jake (교과서 등장 이름 그대로 사용 금지)
- **소재 교체**: 스포츠·음식·직업 등 구체적 어휘 변경
- **수치·장소 교체**: 숫자, 요일, 나라명 등 변경

```
❌ 원본: He (am / is) my teacher.
❌ 부족: She (am / is) my teacher.          (대명사만)
✅ 충분: She (am / is) our coach.           (주어 + 목적어)

❌ 원본: When Paul came home, his sister was watching television.
❌ 부족: When Jake came home, his sister was watching television.  (이름만)
✅ 충분: When Jake got home, his brother was playing video games.
```

## 예문 출처 방침

교과서 참고 예문과 Claude 자체 창작을 **약 50:50**으로 섞는다.

| 출처 | 설명 | 대조 검증 |
|------|------|----------|
| 교과서 참고 변형 | 기본AB·어법드릴·심화·교과서 본문 소재 참고 후 변형 | 필요 (Grep 대조) |
| Claude 자체 창작 | 문법 포인트·수준에 맞게 직접 작성 | 불필요 |

교과서 참고 시 대조 검증:
1. 해당 Lesson md 파일 Read 후 핵심 구절(동사+목적어, 주요 명사) Grep 검색
2. 매칭되면 → 주어·목적어·부사구를 **최소 2개 더** 바꿔서 재작성

## 중2 기초 워크시트 난이도 방침

교과서 **"Let's use it"** 수준. **학습유형 60% + 연습유형 40%** 비율.

| 구분 | 설명 | 문제 유형 |
|------|------|----------|
| 학습유형 (60%) | 괄호 안에서 고르기, 문법 개념 바로 적용 | MCQ 4지선다 |
| 연습유형 (40%) | 빈칸 채우기, 단어 배열 | input · scramble |

기초에서 내는 것 / 내지 않는 것:

| 수여동사 | 주격 관계대명사 |
|---------|----------------|
| ✅ IO(목적격 대명사) 고르기 | ✅ who/which/that 고르기 |
| ✅ IO+DO 어순 맞는 것 고르기 | ✅ 빈칸에 who/which/that 쓰기 |
| ✅ to/for MCQ 인식 문제(3~4문제 소량) | ✅ 두 문장 → 관계대명사로 합치기 (간단) |
| ❌ IO+DO ↔ DO+전치사+IO 전환 → **심화에서만** | ❌ 복잡한 선행사 판단 (고난도) |
| ❌ to/for/of 전치사 전체 문장 변환 → **심화에서만** | |

## 심화 워크시트 표준 구조

**2탭 구조, TOTAL=25, SECTOTALS={a:15, b:10}**

| 탭 | 이름 | 문제 수 | 세부 구성 |
|----|------|---------|-----------|
| Tab 1 | ⚡ 종합 훈련 | 15문제 | 문맥파악MCQ(4) + 오류수정(4) + 문장전환(3) + 우리말영작(4) |
| Tab 2 | 📖 짧은 지문 | 10문제 | Passage A(5) + Passage B(5) |

- **q-type 뱃지**: `ctx`(문맥파악/지문), `fix`(오류수정), `trans`(문장전환), `kor`(우리말영작)
- **kor-hint**: 우리말영작 문제 q-text 안에 `<span class="kor-hint">🇰🇷 한국어</span>` 포함
- **JS**: `var TOTAL=25,SECTOTALS={a:15,b:10},secC={a:0,b:0},secW={a:0,b:0};`
- **억지 유형 금지**: 어색한 유형 대신 학습에 더 효과적인 유형으로 대체. 문제 수는 줄이지 않는다.

## 문제 다양성 원칙 (점진적 구성) — 기초·심화 공통 필수

단순 drill 문제만 나열하는 것은 **초기 도입부에만** 허용. 기초·심화 모두 아래 세 유형을 반드시 섞는다.

| 단계 | 유형 | 설명 | 비율 목표 |
|------|------|------|----------|
| 1 | **문법 drill** | 빈칸 채우기, 형태 선택 (to+V vs V-ing 등) | 기초 약 60%, 심화 약 30% |
| 2 | **구분 문제** | 두 용법 중 어느 쪽인지 식별, 올바른 문장 고르기 | 기초 약 25%, 심화 약 40% |
| 3 | **문맥·해석 문제** | 한국어 해석 선택, 지문 속 용법 찾기, 우리말 영작 | 기초 약 15%, 심화 약 30% |

**단순 패턴 금지 체크:**
- "to만 찍으면 정답"이 되는가? → to-inf 문제에 gerund-correct 선택지 섞기
- "동사 형태만 바꾸면 정답"이 되는가? → 용법 구분·해석 선택 문제 추가
- 모든 문제의 정답이 동일한 패턴(예: 무조건 긍정형)인가? → 부정·의문 문형 혼합
- **한 섹션에서 동일 패턴 정답 70% 이상이면 반드시 다른 유형 추가**

**input 문제 스캐폴딩 기준:**
- 짧은 답(1~2단어): 문장에서 주어+동사 이상 제공 — `She ___ a book.` ✅ / `___ ___ book.` ❌
- 긴 답(절 전체): q-sub로 힌트 단어 또는 두 문장 반드시 제공
- 기초에서 힌트 없이 절 전체 쓰기 금지

**구분 문제 예시 (to-inf 단원):**
```
MCQ: 아래 중 목적을 나타내는 to부정사 문장은?
① She wants to travel to Paris.         ← to-inf 목적어
② He saved money to buy a new phone.    ← 목적의 to-inf ✅
③ I decided to join the team.           ← to-inf 목적어
④ They planned to finish early.         ← to-inf 목적어
```

## 기본 유형 배분

| 유형 | 설명 | 주요 활용 상황 |
|------|------|---------------|
| `mcq-4` | 4지선다 | 문맥 파악이 핵심 (because절 고르기, 형용사vs부사 등) |
| `input-form` | 힌트 주어진 형태 완성 | 주어-동사 수일치, 동사 형태 변환 |
| `input-comp` | 절 전체 영작 | because·when 절 쓰기 — **기초에서는 2문제 이하** |
| `scramble` | 단어 칩 배열 | 어순 정리, 혼합 복습 |

- Section A (Point A): `mcq-4` + `input-form` 조합 — **drill만 나열 금지, 구분·해석 문제 최소 3문제 포함**
- Section B (Point B): 자연스러운 유형만. 억지 유형(접속사 한 글자 채우기, 2지선다 etc.) 금지
- Section C: `scramble` — 단원 혼합 복습

## 심화 유형

| 변형 유형 | 워크시트 타입 |
|-----------|--------------|
| 오류 수정 (틀린 형태 → 고쳐 쓰기) | `input` |
| 문장 변환 (긍정→부정, 현재→과거 등) | `input` |
| 두 문장 합치기 (접속사 활용) | `scramble` |
| 우리말 → 영작 | `input` / `scramble` |

### 두 문장 합치기 — 점진적 스캐폴딩 (4단계)

| 단계 | 제공 범위 | 빈칸 범위 |
|------|-----------|-----------|
| 1단계 | 주절 + 관계대명사 제공 | 관계절 동사+나머지만 빈칸 |
| 2단계 | 주절 + 관계대명사 제공 | 관계절 동사+나머지만 빈칸 (사람/사물 구분) |
| 3단계 | 주절만 제공 | 관계대명사부터 전체 빈칸 |
| 4단계 | 두 문장만 q-sub에 제공 | 완성 문장 전체 빈칸 |

### 심화 문제 문법 용어 지양

문제 본문에 IO · DO · 수여동사 등 문법 메타 용어 쓰지 않는다.

```
❌ 나쁜 예: 「수여동사 + DO + 전치사 + IO」 형태로 바꾸세요.
✅ 좋은 예: 같은 의미가 되도록 빈칸을 채우세요. (to 사용)
```

## 새 워크시트 추가 시 체크리스트

0. **예문 출처 결정**: 교과서 참고(~50%) + 창작(~50%). 교과서 참고 시에만 Grep 대조 검증.
0-1. **단순 패턴 점검**: "한 가지 형태만 찍으면 정답"이 되는 구간 없는지 확인 → 구분·해석 문제 섞기
1. `index.html` 단원 카드 추가 (stats 숫자 업데이트)
2. 정답 비교에 `norm()` 함수 반드시 포함
3. `checkBlank` / `checkArrange` 빈 입력 방지 코드 반드시 포함
4. 모든 해설에 `📝 해석:` 한국어 전체 문장 번역 포함
4-1. **문제 작성 시 "이 해석이 답을 결정하는가?" 즉시 판단 → 필요하면 `.q-kor` 추가**

| q-kor 추가 | q-kor 생략 |
|-----------|-----------|
| 어휘가 생소하거나 문장이 길어 의미 파악이 필요할 때 | q-text가 이미 한국어인 경우 |
| because절·문맥 선택 — 의미를 모르면 답을 고를 수 없을 때 | scramble 📌 형태 (q-text 자체가 한국어) |
| 빈칸 input — 문장 전체 의미를 알아야 정답을 쓸 수 있을 때 | 해석 선택 MCQ (선택지 자체가 한국어) |
| | 순수 어형 변환 (주격→목적격, 동사 활용형 등) — 의미보다 형태가 핵심 |
| | "어법에 맞는 것 고르기" 어순 문제 — 문법 판단이 핵심, 해석이 답을 줘선 안 됨 |
| | 구분 문제 — 해석 제공이 정답 노출이 되는 경우 |

핵심 질문: **"해석을 보면 답이 바로 보이는가?"** → YES이면 생략 검토, NO이면 추가
5. s0 문제 위 한국어 요약 div 삽입 금지
6. 짧은 지문은 ctx 박스 + 빈칸 입력 방식 (TOEIC 형식 금지)
7. 단어는 문항당 vocab 뱃지 또는 ctx 박스로 제공
8. `score-popup.js` 뒤에 `<script src="../sounds.js"></script>` 추가 (효과음)
9. **배열 문제 단어 칩 순서는 정답 순서와 반드시 다르게 배치**
10. **파일 작업 완료 후 자동으로 `git add → commit → push origin main`** — 워크트리 브랜치에서도 `git push origin HEAD:main`으로 main에 바로 머지·배포 완료.
11. **문제 예문 출처 & 유형 규칙은 `worksheet-sentence-guide.md` 참고** (필수)
12. **블로그 배포 자료 — `blog-deploy/<unitNN-topic>.md` 파일로 저장** (신규 워크시트에만 해당, 기존 파일 수정 시 작성하지 않음)
